Выступай в роли системы построения онтологий. 
На вход подается список научных сущностей, каждая из которых имеет:
- название (term)
- определение (definition)
- тип (type)

Твоя задача — определить все возможные семантические связи между сущностями, опираясь 
на их определения, типы и общеизвестные научные зависимости.

Возможные типы отношений (используй только эти):
- "is_a" — A является частным случаем B (наследование, обобщение/специализация)
- "part_of" — A является частью B (структурная связь)
- "depends_on" — A зависит от B или требует B для функционирования
- "uses" — A использует B как компонент, метод или инструмент
- "based_on" — A основан на B (теоретически или вычислительно)
- "improves" — A улучшает B или является его модификацией
- "similar_to" — A похож на B по функции или определению
- "contrasts_with" — A противопоставляется B, является альтернативой

Требования:
1. Основа для каждой связи — только определения сущностей и их типы.
2. Не придумывай новые факты; используй только очевидные семантические зависимости.
3. Не создавай круговых отношений без необходимости.
4. Не создавай дублирующих связей (например, A→B если такая же уже есть).
5. Формат вывода — JSON массив:

{{
    "relations": [
      {{
        "from": "<term A>",
        "to": "<term B>",
        "relation": "<relation_type>",
        "reason": "<краткое объяснение, на основании чего сделан вывод>"
      }},
      ...
    ]
}}

Вывод должен содержать только JSON без комментариев.

Теперь определи связи между следующими сущностями:

{terms_with_definitions_and_types}
